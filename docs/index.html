Features The Quest (v3) Usage
Version 1.3.2

Inject Configurations into UI Schema Easily

A dynamic, lightweight utility built to automatically discover, load, and forge column configurations inside your project's target JSON interfaces.

npm i kschema-fs-ui-alter-config 📋

Features Built for Automation

Designed around efficiency, isolated version control, and convention-based execution.

🔍

Automated Discovery

Scans the filesystem recursively to find all showAll.json configuration files buried in configs directories.

⚙️

Config Ingestion

Loads master schema tables and extracts their columnsConfig properties cleanly with zero hassle.

⚡

In-place Mutation

Updates destination configurations instantly without corrupting structure or other options.

📦

Isolated Versioning

Allows adding new execution pipelines under isolated folders (e.g. v2, v3) dynamically loading the latest version.

The v3 Code Story

In version 3, our code describes an adventure quest using fun module roles, completely avoiding hard-to-read comments.

🕵️

1. The Scout

Goes out to search the realm (directory tree) to locate target configurations (showAll.json inside configs folders).

bin/v3/adventure/scout.js
🔮

2. The Oracle

Reads the ancient master configuration scroll and extracts the correct column definitions layout.

bin/v3/adventure/oracle.js
⚒️

3. The Blacksmith

Melt and forge/transmute the discovered configurations, writing the new definitions in-place.

bin/v3/adventure/blacksmith.js

Programmatic Usage

Integrate columns mutation into your building blocks or validation scripts.

example.js
import load from "kschema-fs-ui-alter-config";
import path from "node:path";

// Execute configuration sync
load({
    toPath: path.join(process.cwd(), "doctors"),
    configPath: path.join(process.cwd(), "Config", "Schemas", "doctors.json"),
    inAction: "Crud"
});
